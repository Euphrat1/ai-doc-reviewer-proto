## Техническое задание 

### 1. Цель
Локальное Windows-приложение для анализа комплекта материалов с помощью LLM. Загрузка материалов из указанной папки, локальная маскировка персональных данных через LM Studio, затем отправка задания в OpenRouter с промтом задачи, запросом оценкой уверенности и генерацией уточняющих вопросов. Возможность отвечать на дополнительные вопросы и повторную обработку задачи. Полный просмотр промпта перед отправкой
Код, структура проекта, и использование библеитек должны быть миниальны и понятным начинающему программисту

### 2. Функциональные требования

**F1. Загрузка материалов**
- Пользователь вводит путь к папке, кнопка «Обзор» через системный диалог 
- Приложение рекурсивно собирает все файлы с расширениями: `.txt`, `.md`, `.csv`, `.xlsx`, `.png`, `.jpg`, `.xml`, `.bpmn`, `.drawio`.
- Текстовые файлы и извлечённый из XML-диаграмм текст отображаются в интерфейсе как контент, доступный для просмотра. Изображения показываются как картинки (превью).

**F2. Маскировка персональных данных (локально)**
- По кнопке «Скрыть личные данные» текстовые материалы отправляются в локальную LLM (LM Studio, модель Mistral-7B-Instruct, температура 0) с жёстким промптом на поиск и замену PII (имена, телефоны, email, адреса, паспортные данные) маской `[MASKED]`.
- Ответ LLM замещает исходный текст в интерфейсе. Изображения и данные таблиц не обрабатываются.
- Пользователь может просмотреть отправляемый в локальную модель промпт (см. F7).

**F3. Отправка задания на анализ (OpenRouter)**
- Пользователь вводит произвольный текст задания.
- К заданию автоматически добавляется системная инструкция:
  * Требуется дать развёрнутый ответ.
  * Оценить уверенность в процентах (1–100).
  * Если уверенность **<50%** — перед ответом сформулировать **4** уточняющих вопроса.
  * Если уверенность **<65%** — **3** вопроса.
  * Если уверенность **<80%** — **2** вопроса.
  * Если уверенность **≥80%** — вопросы не добавляются (остаётся только ответ).
- Перед отправкой пользователь может нажать «Просмотреть промпт» и увидеть полный текст, уходящий в OpenRouter (включая системные инструкции, содержимое материалов и задание). Возможности редактирования промпта в интерфейсе не предусмотрено (можно скопировать и вставить обратно при необходимости).

**F4. Обработка ответа**
- Приложение отображает:
  * Текст ответа,
  * Уверенность в процентах,
  * Блок уточняющих вопросов (если сгенерированы). Каждый вопрос выводится в отдельном поле с возможностью ввести ответ и прикрепить один дополнительный файл (любого поддерживаемого типа) к этому ответу.

**F5. Ручной запрос уточняющих вопросов**
- Вне зависимости от уверенности доступна кнопка «Запросить уточняющие вопросы».
- По нажатию формируется специальный запрос к OpenRouter (с исходными материалами и заданием), в котором LLM просят сгенерировать 1 наиболее полезный уточняющих вопроса для повышения качества итогового анализа. Ответ обрабатывается аналогично F4.

**F6. Финальный анализ с уточнениями**
- После заполнения ответов на автоматические или ручные вопросы (и прикрепления файлов) пользователь нажимает «Отправить уточнения».
- Приложение формирует комбинированный запрос, включающий:
  * исходные материалы,
  * исходное задание,
  * упорядоченный список вопросов и ответов пользователя (с содержимым прикреплённых файлов).
- Этот запрос отправляется в OpenRouter с целью получения итогового анализа. Ответ отображается без дальнейшей генерации вопросов.

### 3. Технические ограничения и стек
- **ОС:** Windows 10/11.
- **Python 3.11+**, виртуальное окружение, зависимости: `streamlit`, `openai`, `pandas`, `openpyxl`, `Pillow`.
- **Запуск:** одной командой `streamlit run app.py`.
- **Локальная LLM:** LM Studio (скачать с https://lmstudio.ai/), модель Mistral-7B-Instruct-v0.3 (Q4_K_M GGUF)
- **Внешний API:** OpenRouter, модель по умолчанию `openai/gpt-4o-mini` (можно изменить в настройках). 

### 4. Упрощённая схема интерфейса (Streamlit)
1. **Боковая панель:** ввод OpenRouter API-ключа (сохраняется в сессии), выбор модели, поле пути к папке материалов + кнопка «Загрузить».
2. **Основная область (вкладки/секции):**
   - *Материалы:* просмотр загруженных текстов и изображений.
   - *Маскировка:* кнопка «Скрыть личные данные», чек-бокс «Показать промпт».
   - *Анализ:* текстовое поле «Задание», кнопка «Анализировать» + «Показать промпт». Блок отображения ответа, уверенности, вопросов.
3. **Блок вопросов:** для каждого вопроса – поле ввода ответа, кнопка прикрепления файла.
4. **Управление:** кнопка «Запросить уточняющие вопросы» (всегда активна после первой загрузки материалов), кнопка «Отправить уточнения» со своим чек-боксом предпросмотра промпта.

### 5. Обработка файлов (детали)
Требуется парсинг всех заявленных типов файлов для передачи в LLM аналитики OpenRouter
  - Результат парсинга отображается как текстовый блок и прикрепляется к запросам анализа.

### 6. Безопасность и гигиена
- Нет аутентификации, многопользовательского режима. Локальная LLM не требует аутентификации.
