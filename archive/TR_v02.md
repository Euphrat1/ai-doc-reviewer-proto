## Техническое задание (v0.2)

### 1. Цель
Локальное Windows-приложение для анализа комплекта материалов с помощью LLM.

Пользователь выбирает папку с материалами. Приложение:
- Загружает и парсит материалы в текстовый корпус.
- (Опционально) выполняет OCR изображений и PDF-страниц (локально).
- Выполняет локальную маскировку PII в текстовых материалах через LM Studio.
- Отправляет задание и материалы в OpenRouter для анализа, запрашивает оценку уверенности и (при необходимости) уточняющие вопросы.
- Позволяет отвечать на вопросы, прикладывать дополнительные файлы и выполнять финальный повторный анализ.
- Всегда даёт возможность просмотреть полный промпт перед отправкой (без редактирования, но с возможностью копировать).

Требование к реализации: код, структура проекта и использование библиотек должны быть минимальны и понятны начинающему программисту.

### 2. Функциональные требования

#### F1. Загрузка материалов
- Пользователь вводит путь к папке или выбирает через кнопку «Обзор» (системный диалог).
- Приложение рекурсивно собирает все файлы с расширениями:
  - `.txt`, `.md`, `.csv`, `.xlsx`, `.xml`, `.bpmn`, `.drawio`, `.png`, `.jpg`, `.pdf`.
- Текстовые материалы и результаты парсинга (см. F1.2) отображаются как просматриваемый текст.
- Изображения отображаются в интерфейсе как превью.

#### F1.2. Парсинг материалов в корпус для LLM (качество ответа)
Для каждого файла формируется **нормализованный текстовый блок** в двух слоях:
- **STRUCTURE**: компактная структура/метаданные, чтобы LLM быстро понимала “что внутри”.
- **EVIDENCE**: фрагменты/выборки, близкие к исходнику, чтобы LLM могла опираться на конкретные факты.

Блок каждого файла обязан начинаться с заголовка:
`FILE: <relative_path> | TYPE: <ext> | SIZE: <bytes> | PARSE: ok|warn|fail`

Минимальные правила по типам:
- `.txt`, `.md`:
  - STRUCTURE: оглавление по заголовкам (если применимо).
  - EVIDENCE: текст с сохранением списков/абзацев.
- `.csv`, `.xlsx`:
  - STRUCTURE: листы (для xlsx), размерность, заголовки колонок, оценка типов колонок (строка/число/дата), доля пустых.
  - EVIDENCE: первые N строк + N “примеров” (например, строки с нетипичными/крайними значениями).
- `.xml` (обычный):
  - STRUCTURE: корневой элемент, частые дочерние элементы, важные атрибуты (id/name/type/status и т.п.).
  - EVIDENCE: извлечённые текстовые узлы/атрибуты (без координат/стилей/шума).
- `.bpmn`:
  - STRUCTURE: participants/pools, lanes, tasks, events, gateways, sequence/message flows.
  - EVIDENCE: текстовая “трасса” процесса шагами (включая условия gateway при наличии).
- `.drawio`:
  - STRUCTURE: страницы, список узлов (id, label), список связей (from→to, label).
  - EVIDENCE: все подписи/labels и edge-list связей (без координат и style-атрибутов).
- `.png`, `.jpg`:
  - Без OCR: только превью в UI и метаданные в STRUCTURE.
  - С OCR (см. F2.1): EVIDENCE включает OCR-текст.
- `.pdf`:
  - STRUCTURE: число страниц, (если доступно) оглавление/закладки.
  - EVIDENCE: извлечённый текст по страницам; если текстового слоя недостаточно — OCR по страницам (см. F2.2).

Важно: корпус обязан иметь **ограничение объёма**. При превышении лимита контента применяется сжатие (например: STRUCTURE всегда целиком, EVIDENCE — выборка/обрезка по страницам/строкам).

#### F2. OCR (локально, вариант 1 — Tesseract)
OCR — опциональная функция для улучшения качества, выполняется **локально**.

##### F2.1. OCR для изображений `.png`, `.jpg`
- В интерфейсе есть настройка: `OCR изображений: выкл/вкл`.
- При включении, каждое изображение проходит OCR и результат добавляется в корпус как EVIDENCE-блок:
  - `OCR_TEXT:` с пометкой языка и базовыми метаданными (размер, имя файла).
- OCR выполняется до маскировки PII (то есть OCR-текст тоже может быть замаскирован в F3).

##### F2.2. OCR для `.pdf`
- В интерфейсе есть настройка: `OCR PDF: выкл/вкл`.
- Алгоритм:
  - Сначала попытка извлечь текстовый слой PDF.
  - Если на странице мало текста (порог задаётся в настройках), страница считается “сканом” и для неё выполняется OCR.
- В корпус добавляется EVIDENCE:
  - `PDF_TEXT p.<n>:` (для извлечённого текста)
  - `PDF_OCR p.<n>:` (для OCR-результата)

##### F2.3. Настройки OCR
- Язык OCR: по умолчанию `rus+eng` (изменяемо).
- Лимиты: максимум страниц PDF для OCR, максимум изображений для OCR (настраиваемо).
- Режим качества: `быстро/качественно` (влияет на предобработку и параметры OCR).

#### F3. Маскировка персональных данных (локально)
- По кнопке «Скрыть личные данные» все **текстовые материалы корпуса** (включая OCR-результаты) отправляются в локальную LLM (LM Studio, Mistral-7B-Instruct, температура 0).
- Используется жёсткий промпт на поиск и замену PII (имена, телефоны, email, адреса, паспортные данные) на маску `[MASKED]`.
- Ответ LLM замещает исходный текст корпуса в интерфейсе.
- Пользователь может просмотреть отправляемый промпт (см. F7).

#### F4. Отправка задания на анализ (OpenRouter)
- Пользователь вводит произвольный текст задания.
- К заданию автоматически добавляется системная инструкция:
  - Требуется дать развёрнутый ответ.
  - Оценить уверенность в процентах (1–100).
  - Если уверенность **<50%** — перед ответом сформулировать **4** уточняющих вопроса.
  - Если уверенность **<65%** — **3** вопроса.
  - Если уверенность **<80%** — **2** вопроса.
  - Если уверенность **≥80%** — вопросы не добавляются (остаётся только ответ).
- Перед отправкой пользователь может нажать «Просмотреть промпт» и увидеть полный текст (системные инструкции + корпус материалов + задание).
- Редактирование промпта в интерфейсе не предусмотрено (можно скопировать и вставить обратно при необходимости).

#### F4.1. Формат ответа LLM (строго для парсинга)
Во всех запросах к OpenRouter LLM должна возвращать **строго JSON** (без Markdown-кода, без пояснений вне JSON).

Формат для первичного анализа (F4):
- `answer` (string): развёрнутый ответ.
- `confidence` (integer): 1–100.
- `questions` (array of strings): список уточняющих вопросов (может быть пустым).

Формат для ручного запроса уточняющего вопроса (F6):
- `questions` (array of strings): массив длины 1.

Формат для финального анализа (F7):
- `answer` (string): итоговый развёрнутый ответ.
- `confidence` (integer): 1–100 (для отображения; вопросы не генерируются).

Если JSON распарсить не удалось, приложение:
- делает 1 автоматическую попытку “исправить формат” (reformat-to-json) отдельным запросом к OpenRouter,
- при повторной неудаче показывает сырой ответ и сообщение об ошибке парсинга.

#### F5. Обработка ответа
- Приложение отображает:
  - текст ответа,
  - уверенность в процентах,
  - блок уточняющих вопросов (если сгенерированы).
- Каждый вопрос выводится отдельным полем с возможностью:
  - ввести ответ,
  - прикрепить один дополнительный файл (любого поддерживаемого типа) к этому ответу.

#### F6. Ручной запрос уточняющих вопросов
- Независимо от уверенности доступна кнопка «Запросить уточняющие вопросы».
- По нажатию формируется запрос к OpenRouter (с корпусом и заданием) с просьбой сгенерировать **1** самый полезный уточняющий вопрос для повышения качества итогового анализа.

#### F7. Финальный анализ с уточнениями
- После заполнения ответов (и прикрепления файлов) пользователь нажимает «Отправить уточнения».
- Приложение формирует комбинированный запрос:
  - корпус материалов,
  - исходное задание,
  - упорядоченный список вопросов и ответов пользователя (с содержимым прикреплённых файлов).
- Запрос отправляется в OpenRouter для получения итогового анализа.
- Итоговый ответ отображается **без дальнейшей генерации вопросов**.

#### F8. Предпросмотр промпта
- Для локальной маскировки и для OpenRouter всегда доступен предпросмотр полного промпта (read-only).

### 3. Технические ограничения и стек
- **ОС:** Windows 10/11.
- **Python 3.11+**, виртуальное окружение.
- **Запуск:** одной командой `streamlit run app.py`.
- **Зависимости (база):** `streamlit`, `openai`, `pandas`, `openpyxl`, `Pillow`.
- **OCR (Tesseract):**
  - Требуется установленный **Tesseract OCR** в системе Windows и доступный в PATH (или указание пути в настройках приложения).
  - Языковые пакеты: `rus`, `eng`.
- **Локальная LLM:** LM Studio, модель Mistral-7B-Instruct-v0.3 (Q4_K_M GGUF).
- **Внешний API:** OpenRouter, модель по умолчанию `openai/gpt-4o-mini` (изменяемо в настройках).

### 4. Упрощённая схема интерфейса (Streamlit)
1. **Боковая панель:**
   - ввод OpenRouter API-ключа (сохраняется в сессии),
   - выбор модели,
   - путь к папке материалов + кнопка «Загрузить»,
   - настройки OCR:
     - `OCR изображений (png/jpg): выкл/вкл`,
     - `OCR PDF: выкл/вкл`,
     - язык OCR (по умолчанию `rus+eng`),
     - лимит страниц PDF для OCR,
     - лимит изображений для OCR,
     - `быстро/качественно`.
2. **Основная область (вкладки/секции):**
   - *Материалы:* просмотр загруженных текстов/корпуса и изображений (превью).
   - *OCR (опционально):* запуск OCR (если включено), просмотр результатов.
   - *Маскировка:* кнопка «Скрыть личные данные», чек-бокс «Показать промпт».
   - *Анализ:* поле «Задание», кнопка «Анализировать» + «Показать промпт». Блок ответа/уверенности/вопросов.
3. **Блок вопросов:** для каждого вопроса — поле ответа + прикрепление файла.
4. **Управление:** «Запросить уточняющие вопросы», «Отправить уточнения» + предпросмотр промпта.

### 5. Безопасность и гигиена
- Нет аутентификации и многопользовательского режима.
- OCR и маскировка выполняются локально.
