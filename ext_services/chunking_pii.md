## Chunking спецификация для PII Masking Service (local_llm) v0.1

Краткий контракт chunking/overlap/merge для `method_id=local_llm`.
Цель: предсказуемое маскирование больших текстов + **глобальная стабильность масок** в рамках одного `/mask` или одного `job_id`.

### 1. Термины и инварианты

- **Chunk**: фрагмент текста длиной `chunk_chars`.
- **Overlap**: перекрытие соседних чанков `overlap_chars`.
- **Registry**: словарь `original → {type, mask}` на время одного запроса/job’а.

**Инварианты (обязательные):**
- одинаковый `original` → одинаковая `<TYPE_N>` по всему входу;
- разный `original` → разные маски (даже при одинаковом `TYPE`);
- `N` сквозной и детерминированный в рамках запроса/job’а (с 1);
- `masked_text` использует только маски из итогового `replacements`.

### 2. Нарезка на чанки

- Параметры: `chunk_chars`, `overlap_chars` (MVP: `overlap_chars > 0`).
- Границы: шаг = `chunk_chars - overlap_chars`; первый чанк с начала текста; последний чанк не выходит за конец текста.

### 3. Стратегия обработки чанков и стабильность масок

#### 3.1 Единый registry на запрос/job
Один общий registry на весь запрос/job: новый `original` → новая `<TYPE_N>`, повтор → существующая.

#### 3.2 Обработка перекрытий
Допускаемая MVP-стратегия:
- LLM маскирует каждый чанк независимо;
- merge результатов через общий registry;
- сборка `masked_text`: первый чанк целиком, остальные — только “хвост” без overlap.

### 4. Merge результатов

#### 4.1 Merge `replacements`
- Итоговый `replacements` уникален по `original` (или `(type, original)` если нужно).
- Конфликт типов для одного `original` → выбрать один тип + добавить `warnings`.

#### 4.2 Сборка `masked_text`
- Соблюдать порядок чанков; не дублировать overlap.
- Все маски в `masked_text` соответствуют итоговому registry.

### 5. Минимальные warnings и тест-кейсы (MVP)
`warnings` (если применимо): конфликт типов; overlap слишком мал; аномально много/мало замен.
Тесты: повтор через границу; сущность на границе; два разных значения одного типа; overlap без дублей в `masked_text`.
